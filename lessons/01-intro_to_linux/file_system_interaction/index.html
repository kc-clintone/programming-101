    üõ†Ô∏è Viewing and Manipulating Files in Linux¶

    Once you know how to move around the Linux file system, the next step is learning how to work with files: view, copy, move, and delete them safely using the command line.

    This lesson covers essential commands:

    • cat ‚Äì view file contents
    • less ‚Äì scroll through files
    • cp ‚Äì copy files
    • mv ‚Äì move or rename files
    • rm ‚Äì delete files

    And we‚Äôll include safety tips so you don‚Äôt lose important data by accident.

    üëÅÔ∏è Viewing File Contents¶

    1. cat ‚Äì View the whole file¶

    cat filename.txt
````

* Displays the entire file in the terminal at once.
* Great for short files.

> ‚ö†Ô∏è For large files, `cat` will flood your screen. Use `less` instead.

---

### 2. `less` ‚Äì Scroll through file contents

```bash
less filename.txt
    • Opens a scrollable viewer in the terminal.

    • Use:

    • ‚Üë and ‚Üì or PageUp / PageDown to navigate.

    • q to quit.

    üß† Analogy: Think of cat as unrolling an entire scroll at once, while less gives you a window to read page by page.

    üìÑ Copying, Moving, and Renaming Files¶

    3. cp ‚Äì Copy a file¶

    cp original.txt copy.txt
    • Creates a duplicate of the file.

    To copy into a folder:¶

    cp notes.txt Documents/

    4. mv ‚Äì Move or rename a file¶

    mv oldname.txt newname.txt
    • Renames a file if destination is a filename.
    • Moves a file if destination is a directory.

    Example ‚Äì Move to another folder:¶

    mv notes.txt Archive/

    üß† Tip: Unlike cp, mv removes the original file after moving.

    üóëÔ∏è Deleting Files (Carefully)¶

    5. rm ‚Äì Remove a file or directory¶

    rm file.txt

    ‚ö†Ô∏è There‚Äôs no Recycle Bin. Deleted means gone.

    ‚ö†Ô∏è Safety Tip: Use -i (interactive)¶

    rm -i file.txt
    • Asks for confirmation before deleting.

    Even safer for folders:¶

    rm -ri myfolder/
    • -r means recursive (needed to delete folders)
    • -i prompts for each file inside

    üåü Bonus Tips: Wildcards & Safety¶

    üêö Use Wildcards to Select Multiple Files¶

    rm *.log
    • Deletes all .log files in the current directory.
    cp *.txt backup/
    • Copies all .txt files to backup/

    Wildcard symbols:¶

    Symbol Meaning
    * Any number of characters
    ? One single character

    üîí Avoiding Data Loss¶

    • üõë Never run rm -rf / ‚Äî it will delete your whole system.
    • ‚úÖ Use rm -i for important deletions.
    • üíæ Always double-check the file or folder name.
    • üì¶ Consider using trash-cli or mv to a Trash/ directory if unsure.

    üß™ Try This:¶

    touch test.txt

cat test.txt
cp test.txt backup.txt
mv backup.txt archive.txt
rm -i archive.txt

    Each command here demonstrates file creation, viewing, copying, renaming, and deletion ‚Äî interactively and safely.

    üéØ Quick Reference¶

    Command Action
    cat View short file contents
    less View and scroll through files
    cp Copy files
    mv Move or rename files
    rm Delete files
    -i Prompt before delete/copy
    * / ? Wildcard match for files
